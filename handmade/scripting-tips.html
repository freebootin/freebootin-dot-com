Scripting Tips

Preface

Every computer user should know how to do basic scripting. It increases your productivity by an order of magnitude and eliminates the need for many large programs. I would estimate that with just the unix core utilities and cURL you could script just about anything.

Use the Google Shell Style Guide

Regardless of your opinion on Google, their shell style guide is very will done. You should read it and apply it to every script you write. Having a consistant way of writing scripts will help make your scripts easier to read when you come back to them a year later. Also, it will help you to avoid some of the gatcha's of shell scripting.

Google Shell Style Guide

Use ShellCheck

From the ShellCheck github:

ShellCheck is a GPLv3 tool that gives warnings and suggestions for bash/sh shell scripts.

The goals of ShellCheck are:

  • To point out and clarify typical beginner's syntax issues that cause a shell to give cryptic error messages.
  • To point out and clarify typical intermediate level semantic problems that cause a shell to behave strangly and counter-intuitively.
  • To point out subtle caveats, corner cases and pitfalls that may cause an advanced user's otherwise working script to fail under future circumstances.

Every script you write should be run throught ShellCheck. Not only will it catch a lot of bugs for you, it will also make you a better shell programmer.

You can get ShellCheck through your package manager or from github.

Tutorials

Besides the Google Shell Style Guide there have not been any tutorials that I have ever found useful. Rather than using tutorials I have found it better to just read the man pages or read other peoples scripts. I can't recommend tutorials in general.

Should You Use ZSH?

Probably not. Bash is the default shell all of the main Linux distributions and as such you are guarenteeed to encounter it frequently. You also won't find scripts being written in zsh and it is not supported by shellcheck.
